### Added
- Canonical `detail::ArenaAddress<AT>` / `detail::ConstArenaAddress<AT>` address layer
  (issue #375). `BasicArenaView` now exposes `granule_offset`, `try_user_ptr`,
  `try_user_idx_from_raw`, `try_block_idx_from_user_idx`, `try_user_idx_from_block_idx`
  alongside the existing `valid_block` / `block` helpers. Manager helpers
  (`make_pptr_from_raw`, `block_raw_ptr_from_pptr`, `raw_user_ptr_from_pptr`,
  `find_block_from_user_ptr`) delegate to this layer instead of duplicating raw arithmetic.
- `ManagerT::try_tree_node<T>(pptr)` (returns `BlockHeader<AT>*` / `nullptr` and sets
  `PmmError::InvalidPointer` on failure) and `ManagerT::tree_node_unchecked<T>(pptr)`
  (returns a `BlockHeader<AT>&` under a strict precondition).
- `pptr<T>::try_tree_node()` and `pptr<T>::tree_node_unchecked()`.
- `scripts/check-source-loc-budget.sh` and `scripts/source-loc-baseline.txt` enforce the
  production LOC budget in CI and pre-commit.
- `tests/test_issue375_address_layer.cpp`, `tests/test_issue375_containers.cpp`,
  `tests/test_issue375_tree_node.cpp`, `tests/test_issue375_static_checks.cpp`.

### Changed
- `parray<T>::ensure_capacity` and `pstring::ensure_capacity` now grow through
  `ManagerT::reallocate_typed<T>` (in-place shrink/grow when possible, fallback
  relocation copies only live elements/bytes).
- `parray<T>::resolve_data` / `free_data` and `pstring::resolve_data` / `free_data`
  go through canonical `pptr<T>::resolve_unchecked` and `ManagerT::deallocate_typed`
  instead of the old `detail::resolve_granule_ptr` raw arithmetic.
- `ManagerT::reallocate_typed<T>` now validates the source `pptr` through the
  shared `try_checked_block_from_pptr` helper before any raw block access.
  Below-header offsets and stale pptrs after `deallocate_typed` are rejected
  with `PmmError::InvalidPointer` instead of dereferencing into the arena.

### Removed
- Sentinel-based `ManagerT::tree_node(pptr)` (the old API silently returned a thread-local
  zeroed `BlockHeader` on invalid pptr).
- Legacy `pptr<T>::tree_node()` ref-returning convenience method.
- `detail::block_at_checked`, `detail::resolve_granule_ptr_checked`,
  `detail::ptr_to_granule_idx_checked` — superseded by `ArenaAddress` methods.
- The internal non-aligned `Block<AT>` fallback in `find_block_from_user_ptr` (Block size
  is statically asserted to be a multiple of granule_size).
